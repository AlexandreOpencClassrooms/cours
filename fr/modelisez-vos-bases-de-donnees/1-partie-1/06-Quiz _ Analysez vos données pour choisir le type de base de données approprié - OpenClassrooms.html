  • Question 1

    Que dire de ce jeu de données ?

    • Il est structuré car il est facilement représentable sous forme de tableau, dans lequel il y a peu de cases vides.

    • Il est structuré car ce n’est ni du texte, ni une image, ni une vidéo

    • Il est peu structuré car toutes les valeurs dans les cases du tableau sont différentes.

    Ces données sont structurées car facilement représentables sous forme de tableau, dans lequel il y a peu de cases vides. Par ailleurs, il existe des données peu structurées qui ne sont ni du texte, ni de l'image ni de la vidéo (exemple : données sous format clé-valeur).

  • Question 2

    En vous basant uniquement sur le nom de ces 3 SGBD fictifs, trouvez lequel d’entre eux serait le mieux adapté pour accueillir ces données.

    • NoSQLBigDataLake

    • PowerSQL

    • VideoStockDB

    NoSQLBigDataLake semble adapté aux données de type Big Data, qui sont souvent peu structurées, et il semble utiliser le NoSQL, qui est lui aussi plus adapté à des données peu structurées. VideoStockDB semble être un SGBD fait pour stocker des vidéos, données peu structurées et qui n’ont rien à voir avec nos données. PowerSQL semble quant à lui utiliser le SQL, langage privilégié pour les données structurées.

  • Question 3

    Supposons que ces données soient stockées dans un fichier de type Excel ou CSV, consultable avec un tableur. Nous souhaitons que plusieurs programmes informatiques puissent accéder et actualiser ces données en vue de se les partager entre eux.
    Quelle serait la solution technique la plus pratique en termes de temps de développement de ces programmes ?

    • Coder les programmes de manière à ce qu’ils puissent ouvrir le fichier et le modifier.

    • Faire autant de copies séparées qu’il y a de programmes informatiques, et les faire interagir indépendamment sur leur propre copie.

    • Entrer ces données dans une base de données relationnelle gérée par un SGBDR.

    Le plus souvent, il est préférable de confier la base de données à un SGBD, qui est un logiciel spécialisé dans l’optimisation du stockage et des accès / modifications de données. Cela évite de devoir recoder dans chaque programme la gestion des données. De plus, un SGBD permet à chacun des programmes d’accéder aux données et de se les partager, ce qui ne serait pas le cas si chaque programme avait sa propre copie.

  • Question 5

    De quel type sont les modélisations données à la question précédente ?

    • Diagramme Entité-Associations (MERISE)

    • Diagramme de classes UML

    • Modélisation relationnelle

    Il s’agit d’un diagramme de classes UML. Il ne s’agit pas d’un modèle relationnel car on y voit les cardinalités (0, 1, *, etc.) associées à chaque association. Aussi, le diagramme E-A de la méthode MERISE est graphiquement différent de l’UML (jetez un coup d'œil au cours pour vous rafraîchir la mémoire ).

  • Question 6

    Que dire des données présentées dans l’énoncé de ce quiz ?

    • Il y a de la redondance car l'information selon laquelle « le film Fleabag est produit par les studios Banijay » est présente à plusieurs endroits

    • Il n’y a pas de redondance car même si l’information selon laquelle « le film Fleabag est produit par les studios Banijay » est présente à plusieurs endroits, il n’y a pas d’incohérence.

    • Il n’y a pas de redondance car aucune information n’est présente en double.

    Qu’il y ait une incohérence dans les données ou pas, une information présente à plusieurs endroits marque une redondance. C’est problématique même en l’absence d’incohérence, car nous ne sommes pas à l’abri qu’une incohérence se glisse plus tard, par exemple si l’information doit être actualisée et que l’utilisateur/utilisatrice oublie d’actualiser l’information à tous les endroits où l’information est présente.

  • Question 9

    1/ Vous voulez créer un réseau social destiné à ceux qui aiment écrire : apprentis poètes, écrivains confirmés, etc. Ils y posteront principalement des textes, de tailles très variables. Ce réseau social devra inclure un moteur de recherche performant permettant de rechercher des mots clés (ou un ensemble de mots clés) parmi tous les textes.

    Quel type de base de données allez vous utiliser ?

    • Une base de données de type SQL

    • Une base de données de type NoSQL

    • Un fichier CSV

    La base de données de votre réseau social contiendra principalement du texte, qui est une donnée peu structurée car elle est difficile à représenter sous forme de tableau. Vous utiliserez donc un SGBD de type NoSQL spécialisé dans le stockage et la recherche de texte.

  • Question 10

    Vous devez programmer un logiciel de comptabilité destiné à concevoir et stocker des factures. Légalement, la structure d’une facture est définie de manière précise : toute facture doit en effet contenir des mentions obligatoires telles que la date d’émission, le numéro de facture, l’identité de l’acheteur, le numéro de TVA, etc.

    Quel type de base de données allez vous utiliser ?

    • Une base de données de type SQL

    • Une base de données de type NoSQL

    • Un fichier CSV

    Vous utiliserez pour les factures une base relationnelle pour des données structurées : la structure des factures est très clairement définie et facile à représenter sous forme de tableau.
